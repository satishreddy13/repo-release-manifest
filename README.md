# Release Manifest Repository

Owned by CM. This repository is the single source of truth for every bi-weekly
sprint release. It does **not** contain application code — it records which
branch/commit from each team repo is in a given build, what COTS version is
included, and what the deployment status of each environment is.

---

## Repository Layout

```
releases/
  sprint-NN/
    manifest.yml        ← build definition (CM edits this)
    RELEASE-NOTES.md    ← auto-generated human-readable report
    cots-sprint-NN.md   ← COTS version record for this sprint

templates/
  manifest.yml.template       ← copy to releases/sprint-NN/ at sprint start
  RELEASE-NOTES.md.template   ← reference only; generated by script
  cots.md.template            ← copy and fill in COTS details each sprint

scripts/
  cut-sprint.sh         ← CM runs at sprint START  — creates sprint branches
  generate-report.sh    ← CM runs at BUILD DAY     — produces RELEASE-NOTES.md
  tag-environments.sh   ← CM runs at DEPLOYMENT    — stamps sit/uat tags
  close-sprint.sh       ← CM runs after UAT passes — merges to main, tags release
```

---

## CM Sprint Runbook

### Sprint Start
```bash
./scripts/cut-sprint.sh sprint-13
```
Creates `sprint/sprint-13` branch in every team repo and scaffolds
`releases/sprint-13/manifest.yml` from the template.

### Build Day (sprint close)
1. Teams confirm which ELM items are merged into their sprint branch.
2. CM fills in commit SHAs and sets `included: true/false` in `manifest.yml`.
3. CM updates the COTS section with the latest version details.
4. CM runs:
   ```bash
   ./scripts/generate-report.sh sprint-12
   ```
   This writes `releases/sprint-12/RELEASE-NOTES.md`.
5. CM commits manifest + report, opens a PR titled `Release sprint-12`.
6. CM deploys each included team's branch to SIT, then runs:
   ```bash
   ./scripts/tag-environments.sh sprint-12 sit
   ```

### SIT Passes → Promote to UAT
```bash
./scripts/tag-environments.sh sprint-12 uat
```

### UAT Passes → Close Sprint
```bash
./scripts/close-sprint.sh sprint-12
```
Merges each included team's sprint branch to `main` and creates
`release/sprint-12` tags across all repos.

---

## Removing a Team Mid-Sprint

**Before build is cut:** Set `included: false` in `manifest.yml`. No branch
changes needed. Team's work defers automatically to next sprint.

**After deployment to SIT:** Create a patch release entry:
```bash
cp -r releases/sprint-12 releases/sprint-12-patch1
# edit manifest.yml — set the problem team to included: false
./scripts/generate-report.sh sprint-12-patch1
./scripts/tag-environments.sh sprint-12-patch1 sit
```

## Branch Naming Conventions (all team repos)

| Branch | Who creates | Purpose |
|--------|-------------|---------|
| `main` | CM | Stable — tagged releases only |
| `sprint/sprint-NN` | CM | Sprint integration branch |
| `feature/ELM-XXXX-short-desc` | Developer | Individual work item |
| `fix/ELM-XXXX-short-desc` | Developer | Bug fix or hotfix |

## Commit Message Convention
```
ELM-1234: Short description of what changed

Optional longer body explaining why.
```
The `ELM-XXXX` prefix links the commit to the IBM ELM work item via the
GitHub Enterprise ↔ ELM integration.

---

## Environment Tag Convention (all team repos)

| Tag | Meaning |
|-----|---------|
| `sit/sprint-NN` | Commit deployed to SIT for sprint NN |
| `uat/sprint-NN` | Commit deployed to UAT for sprint NN |
| `release/sprint-NN` | Commit merged to main after UAT passed |
